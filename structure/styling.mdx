The project uses **Tailwind CSS** with a custom design system built on CSS variables for theming. This approach enables seamless light/dark mode switching and consistent styling across the entire application.

## Tailwind CSS Setup

### Configuration File

**File:** `tailwind.config.js`

```javascript tailwind.config.js:1-76
// tailwind.config.js

const defaultTheme = require('tailwindcss/defaultTheme');
const colors = require('tailwindcss/colors');

module.exports = {
  content: [
    './app/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}',
    './src/**/*.{js,ts,jsx,tsx,mdx}',
  ],
  darkMode: 'class',
  theme: {
    extend: {
      fontFamily: {
        sans: ['var(--font-geist)', ...defaultTheme.fontFamily.sans],
        mono: ['var(--font-geist-mono)'],
      },
      borderRadius: {
        lg: 'var(--radius)',
        md: 'calc(var(--radius) - 2px)',
        sm: 'calc(var(--radius) - 4px)',
      },
      colors: {
        ...colors,
        background: 'hsl(var(--background))',
        foreground: 'hsl(var(--foreground))',
        card: {
          DEFAULT: 'hsl(var(--card))',
          foreground: 'hsl(var(--card-foreground))',
        },
        primary: {
          DEFAULT: 'hsl(var(--primary))',
          foreground: 'hsl(var(--primary-foreground))',
        },
        // ... additional color tokens
      },
      boxShadow: {
        input: '0px 2px 3px -1px rgba(0,0,0,0.1), ...',
      },
    },
  },
  plugins: [require('tailwindcss-animate')],
};
```

## CSS Variables

The design system is built on CSS variables for dynamic theming.

<AccordionGroup>
  <Accordion title="Color Variables - Light Mode">
    ```css
    :root {
      --background: #f5f5f7;    /* Off-white background */
      --foreground: #0a0a0a;    /* Nearly black text */
      
      --main: #166534;          /* Green primary color */
      --accent: #22c55e;        /* Bright green accent */
      
      --surface: #dadada66;     /* Semi-transparent surface */
      --overlay: #131313;       /* Dark overlay */
      --muted: #393939;         /* Muted gray */
      
      --border-default: #a8a8a8; /* Border color */
    }
    ```
    
    Light mode uses a clean, minimal palette with high contrast for readability.
  </Accordion>

  <Accordion title="Color Variables - Dark Mode">
    ```css
    .dark {
      --background: #0a0a0a;    /* Nearly black background */
      --foreground: #f5f5f7;    /* Off-white text */
      
      --main: #166534;          /* Same green primary */
      --accent: #22c55e;        /* Same bright green */
      
      --surface: #13131380;     /* Dark semi-transparent surface */
      --overlay: #dadada;       /* Light overlay */
      --muted: #a2a2a9;         /* Light muted gray */
      
      --border-default: #333333; /* Darker border */
    }
    ```
    
    Dark mode inverts background/foreground while keeping brand colors consistent.
    
    <Note>
      Notice that `--main` and `--accent` (brand colors) remain the same in both themes for consistency.
    </Note>
  </Accordion>

  <Accordion title="Typography Variables">
    ```css
    :root {
      --font-geist: 'Geist Sans', sans-serif;
    }
    ```
    
    Font family is defined as a CSS variable and used in:
    1. **Tailwind config** - `fontFamily.sans`
    2. **Global styles** - `body { font-family: var(--font-geist); }`
    3. **Layout component** - `className={GeistSans.className}`
  </Accordion>

## Theming System

### Dark Mode Implementation

The project uses `next-themes` for theme management:

```json package.json:28
"next-themes": "^0.4.6"
```

**How it works:**

1. **ThemeToggle component** - User interface for switching themes
2. **next-themes provider** - Manages theme state and localStorage
3. **Class-based toggle** - Adds `.dark` class to `<html>` element
4. **CSS variables** - `.dark` selector overrides color values

### Using Variables in Tailwind

Tailwind classes reference CSS variables:

```jsx
<div className="bg-[var(--surface)] border-[var(--border-default)]">
  <p className="text-[var(--foreground)]">
    Content adapts to theme
  </p>
</div>
```

**Real example from the codebase:**

```jsx src/app/page.js:19
<section className="... border-[var(--border-default)] bg-[var(--surface)] ...">
  <AnimatedImgs />
</section>
```

### Theme Transitions

```css src/styles/global.css:41-47
body {
  background: var(--background);
  color: var(--foreground);
  font-family: var(--font-geist);
  transition:
    background 0.5s,
    color 0.5s;
}
```

Smooth 0.5s transitions when switching between light and dark modes.

## Common Styling Patterns

### Card/Section Styling

Most sections use this consistent pattern:

```jsx
<section className="
  mx-5                                    /* Horizontal margin */
  mb-10                                   /* Bottom spacing */
  rounded-[25px]                          /* Large rounded corners */
  border-[1px]                            /* 1px border */
  border-[var(--border-default)]          /* Theme-aware border color */
  bg-[var(--surface)]                     /* Semi-transparent background */
  p-5                                     /* Padding */
  text-[16px]                             /* Font size */
  md:text-[16px]                          /* Responsive text size */
  md:dark:border-[var(--border-default)]  /* Dark mode border on medium+ */
">
  {/* Content */}
</section>
```

### Responsive Design

Tailwind's responsive prefixes:

```jsx
<div className="
  p-2              /* Padding on mobile */
  md:p-4           /* Larger padding on tablet */
  lg:p-6           /* Even larger on desktop */
  
  text-sm          /* Small text on mobile */
  md:text-base     /* Normal text on tablet+ */
">
```

**Breakpoints:**
- `sm:` - 640px+
- `md:` - 768px+
- `lg:` - 1024px+
- `xl:` - 1280px+
- `2xl:` - 1536px+

### Container Width

Consistent max-width across pages:

```jsx src/app/layout.js:107
<div className="container m-auto max-w-[980px]">
  {/* All content constrained to 980px */}
</div>
```

## Styling Checklist

When styling a new component:

- [ ] Use `var(--border-default)` for borders
- [ ] Use `var(--surface)` for section backgrounds
- [ ] Use `var(--foreground)` for text (or rely on inherited color)
- [ ] Apply `rounded-[25px]` for section corners
- [ ] Add responsive classes with `md:` and `lg:` prefixes
- [ ] Include smooth transitions for interactive elements
- [ ] Test in both light and dark modes
