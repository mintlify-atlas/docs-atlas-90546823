The project uses **Next.js 15 App Router** with file-based routing. Every folder in `src/app/` with a `page.js` file automatically becomes a route.

## Route Structure

Here's the complete routing structure:

```
src/app/
├── layout.js                 # Root layout (wraps all pages)
├── page.js                   # Home page (/)
├── not-found.js              # 404 page
├── about/
│   └── page.js               # About page (/about)
├── connect/
│   └── page.js               # Contact page (/connect)
├── made-by-arsh/
│   ├── page.js               # Portfolio listing (/made-by-arsh)
│   ├── templates/
│   │   └── [slug]/
│   │       ├── page.js       # Template detail (/made-by-arsh/templates/[slug])
│   │       └── TemplateContent.js
│   └── web-projects/
│       └── [slug]/
│           └── page.js       # Web project detail (/made-by-arsh/web-projects/[slug])
└── resume/
    └── page.js               # Resume page (/resume)
```

## Dynamic Routes

Dynamic routes use **bracket notation** `[slug]` to create parameterized URLs.

<AccordionGroup>
  <Accordion title="/made-by-arsh/templates/[slug] - Template Details">
    **File:** `src/app/made-by-arsh/templates/[slug]/page.js`
    
    Displays individual template details. The `[slug]` parameter matches the template name.
    
    ```javascript src/app/made-by-arsh/templates/[slug]/page.js:1-21
    // app/made-by-arsh/templates/[slug]/page.js
    
    import { templateData } from '@/data/webTemplatesBase';
    import TemplateContent from './TemplateContent';
    import { notFound } from 'next/navigation';
    
    export function generateMetadata({ params }) {
      const template = templateData.find((t) => t.slug === params.slug);
      return {
        title: `${template.name} | Made by Arsh | Arsh Saxena`,
        description: `${template.name} template made by Arsh. ${template.desc}`,
      };
    }
    
    export default function TemplatePage({ params }) {
      const template = templateData.find((t) => t.slug === params.slug);
      if (!template) return notFound();
      
      return <TemplateContent template={template} />;
    }
    ```
    
    **Example URLs:**
    - `/made-by-arsh/templates/apple-style-navigation-bar`
    - `/made-by-arsh/templates/modern-footer`
    - `/made-by-arsh/templates/simple-old-school`
    
    <Tip>
      The `generateMetadata()` function creates dynamic SEO metadata based on the route parameters.
    </Tip>
  </Accordion>

  <Accordion title="/made-by-arsh/web-projects/[slug] - Web Project Details">

## 404 Not Found Page

**File:** `src/app/not-found.js`

Custom 404 page for invalid routes. This is automatically shown when:
- User navigates to a non-existent route
- A dynamic route calls `notFound()` (e.g., invalid slug)

```javascript
import { notFound } from 'next/navigation';

if (!template) return notFound();
```

## Routing Patterns

### Getting Route Parameters

```javascript
export default function Page({ params }) {
  // params.slug contains the dynamic segment
  const item = data.find(d => d.slug === params.slug);
  return <div>{item.name}</div>;
}
```

### Programmatic Navigation

```javascript
import { useRouter } from 'next/navigation';

function Component() {
  const router = useRouter();
  
  const handleClick = () => {
    router.push('/about');
  };
}
```

### Link Component

```javascript
import Link from 'next/link';

<Link href="/about">About Me</Link>
<Link href={`/made-by-arsh/templates/${slug}`}>View Template</Link>
```

## Sitemap Generation

The project uses `next-sitemap` to automatically generate sitemaps:

**Config:** `next-sitemap.config.js`

```javascript
module.exports = {
  siteUrl: 'https://www.arshsaxena.in',
  generateRobotsTxt: true,
  // Additional configuration
};
```

Sitemaps are generated during the build process:

```json package.json:8
"postbuild": "next-sitemap"
```

Generated files:
- `public/sitemap.xml` - Main sitemap
- `public/sitemap-0.xml` - Sitemap chunk
- `public/robots.txt` - Robots configuration

## Best Practices

<CardGroup cols={2}>
  <Card title="File Colocation" icon="folder-tree">
    Keep route-specific components in the same folder as the page (e.g., TemplateContent.js next to page.js)
  </Card>
  <Card title="Metadata Export" icon="file-code">
    Always export metadata from pages for proper SEO and social sharing
  </Card>
  <Card title="Not Found Handling" icon="circle-exclamation">
    Use notFound() for invalid dynamic routes instead of showing error states
  </Card>
  <Card title="Server Components" icon="server">
    Keep pages as server components by default for optimal performance
  </Card>
</CardGroup>
