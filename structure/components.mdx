The project follows a **feature-based component organization** where components are grouped by their functional area or page context. All components live in `src/components/` and are organized into logical subdirectories.

## Component Patterns

### Client vs Server Components

The project leverages React Server Components (default in App Router) with selective client components:

<CodeGroup>
```jsx Server Component (default)
// No directive needed - server component by default
export default function TextSection() {
  return (
    <section>
      <p>Static content rendered on the server</p>
    </section>
  );
}
```

```jsx Client Component
'use client';

import { useState, useEffect } from 'react';

export default function SkillsTable() {
  const [chunkedSkills, setChunkedSkills] = useState([]);
  // Client-side interactivity
}
```
</CodeGroup>

**Use `'use client'` when:**
- Using React hooks (`useState`, `useEffect`, etc.)
- Adding event listeners
- Accessing browser APIs
- Using third-party libraries that require client-side execution

### File Naming Conventions

<Note>
  The project uses a mix of `.js` and `.jsx` extensions:
  - **Layout components** - `.js` (Navbar.js, Footer.js)
  - **Page sections** - `.jsx` (SkillsTableSection.jsx, AnimatesImgsSection.jsx)
  
  Both extensions work identically in Next.js. The `.jsx` extension explicitly indicates JSX usage.
</Note>

## Assets Organization

### SVG Icons

Skill icons and other SVGs are stored in `src/assets/`:

```
src/assets/about/skills/
├── bootstrap.svg
├── css.svg
├── html.svg
├── javascript.svg
├── nextjs.svg
├── python.svg
├── react.svg
├── tailwindcss.svg
└── ... (24 skill icons total)
```

**Import pattern:**
```jsx src/components/About/SkillsTableSection.jsx:7-10
import CppIcon from '@/assets/about/skills/cpp.svg';
import PythonIcon from '@/assets/about/skills/python.svg';
import NextjsIcon from '@/assets/about/skills/nextjs.svg';
import TailwindcssIcon from '@/assets/about/skills/tailwindcss.svg';
```

<Tip>
  SVGs are imported as React components thanks to `@svgr/webpack` configuration in `next.config.js`.
</Tip>
