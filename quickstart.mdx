Get your Arsh Saxena Portfolio clone up and running in minutes. This guide will take you from cloning the repository to seeing your portfolio live in your browser.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 18.17 or later
- **npm**, **yarn**, or **pnpm** package manager
- **Git** for cloning the repository

<Note>
This portfolio is built with Next.js 15.5.9 and React 19.1.0, featuring a modern tech stack with Tailwind CSS, Framer Motion, and next-themes for dark mode support.
</Note>

## Get started

<Steps>
  <Step title="Clone the repository">
    Clone the portfolio repository to your local machine:

    ```bash
    git clone https://github.com/arshsaxena/arshsaxenadotin.git
    cd arshsaxenadotin
    ```

    The repository structure includes:
    - `src/app/` - Next.js App Router pages
    - `src/components/` - React components
    - `src/data/` - Portfolio content data
    - `src/styles/` - Global styles and CSS variables
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    This will install key dependencies including:
    - **Next.js 15.5.9** - React framework
    - **Tailwind CSS** - Utility-first CSS framework
    - **Framer Motion** - Animation library
    - **next-themes** - Dark mode support
    - **Tabler Icons** & **Lucide React** - Icon libraries
  </Step>

  <Step title="Start the development server">
    Launch the development server to see your portfolio in action:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    You should see output similar to this:

    ```bash
    ▲ Next.js 15.5.9
    - Local:        http://localhost:3000
    - Network:      http://192.168.1.1:3000

    ✓ Ready in 2.3s
    ```
  </Step>

  <Step title="Open in your browser">
    Navigate to [http://localhost:3000](http://localhost:3000) in your web browser to view your portfolio.

    The hot reload feature will automatically refresh the page when you make changes to the source code.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    If port 3000 is occupied, Next.js will automatically try the next available port (3001, 3002, etc.). You can also specify a custom port:

    <CodeGroup>
      ```bash npm
      npm run dev -- -p 3001
      ```

      ```bash yarn
      yarn dev -p 3001
      ```

      ```bash pnpm
      pnpm dev -p 3001
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Module not found errors">
    If you encounter module resolution errors, try these steps:

    1. Delete `node_modules` and lock files:
       ```bash
       rm -rf node_modules package-lock.json yarn.lock pnpm-lock.yaml
       ```

    2. Clear Next.js cache:
       ```bash
       rm -rf .next
       ```

    3. Reinstall dependencies:
       ```bash
       npm install
       ```
  </Accordion>

  <Accordion title="SVG imports not working">
    The portfolio uses `@svgr/webpack` configured in `next.config.js:34-39` to handle SVG imports. If SVG imports fail:

    - Verify `@svgr/webpack` is installed in `devDependencies`
    - Check that the webpack config in `next.config.js` includes the SVG loader rule
    - Restart the development server after configuration changes
  </Accordion>

  <Accordion title="Dark mode not switching">
    The portfolio uses `next-themes` for theme management. If dark mode isn't working:

    - Check that `ThemeProvider` is properly wrapped around your app layout
    - Verify `darkMode: 'class'` is set in `tailwind.config.js:14`
    - Clear browser cache and localStorage
    - Ensure CSS variables are defined for both light and dark modes in your global styles
  </Accordion>

  <Accordion title="Build fails with 'output: export' error">
    The portfolio is configured for static export (`output: 'export'` in `next.config.js:6`). This means:

    - Server-side features like API routes are not supported
    - Dynamic routes must use `generateStaticParams`
    - Image optimization requires `images.unoptimized: true` (already configured in `next.config.js:31-33`)

    If you need server-side features, remove the `output: 'export'` configuration.
  </Accordion>
</AccordionGroup>

<Warning>
When building for production with `npm run build`, the portfolio generates a static site in the `out/` directory. Make sure to test the build locally before deploying.
</Warning>
